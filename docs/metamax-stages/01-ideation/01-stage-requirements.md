# Stage 01 — Ideation: Requirements & Sequence

## Purpose
Define the project vision, scope, success criteria, and high-level approach before any design or technical work begins.

## Required Inputs
- Project name and one-sentence description
- Business or user problem being solved
- Target users / stakeholders
- Any known constraints (budget, timeline, technology, compliance)

## Expected Outputs
- Clear project vision statement
- High-level scope (what is in / out of scope)
- Success metrics (3–5 measurable outcomes)
- Initial risk register
- Recommended project type (Enterprise App, Mobile App, SaaS, Game, etc.)

## Agent Roles
- **Lead Agent**: Orchestrates research and synthesis
- **Research Agent**: Gathers market, competitor, and user insights
- **Synthesis Agent**: Consolidates findings into clear recommendations

## Sequence
1. Project intake & clarification
2. Research & competitive analysis
3. Scope definition workshop (HITL)
4. Success metrics definition
5. Risk identification
6. Project type recommendation + guardrails activation
7. Human approval to proceed to Design stage

## Guardrails Activated
- Project type detection
- External tooling / API key checklist
- Initial scope boundary enforcement
