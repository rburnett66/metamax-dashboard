# Stage 02 — Design: Requirements & Sequence

## Purpose
Translate the approved vision and requirements into a clear, beautiful, and usable product design (UI/UX + visual system).

## Required Inputs
- Approved project vision and success metrics (from Ideation)
- User personas and key user journeys
- Brand guidelines (if any)

## Expected Outputs
- Complete design system (colors, typography, components, spacing)
- High-fidelity wireframes for all key screens
- Interactive prototype
- Design handoff package for development
- Accessibility audit (WCAG 2.1 AA minimum)

## Agent Roles
- **Design Lead Agent**: Owns overall visual direction and component system
- **UX Agent**: Creates user flows, wireframes, and interaction patterns
- **Visual Agent**: Builds the design system, illustrations, and motion guidelines

## Sequence
1. Design system foundation
2. User flow mapping + wireframing
3. High-fidelity screen designs
4. Interactive prototype
5. Design QA + accessibility review
6. Developer handoff
7. Human approval to proceed to Tech Plan
